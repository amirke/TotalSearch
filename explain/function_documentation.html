🔄 Complete Function Flow Diagram

📁 File Operations Flow

1. File Opening Process

  • User clicks "Open File2" button
  • MainWindow::openFile2() called
  • MainWindow::KClean() - Clean previous state
  • MainWindow::KOpenFile() - Start file loading
  • LogDataWorker::startIndexing() - Begin indexing
  • LogDataWorker::doIndexing() - Background indexing
  • LogDataWorker::indexingFinished signal emitted
  • MainWindow::KDisplayFile() - Display file content
  • LogMainView::updateViewport() - Render content

2. Search Operations Flow

  • User clicks "Find in File" button
  • MainWindow::KFindInFile() called
  • SearchDialog opens for configuration
  • LogDataWorker::searchInFileSync() - Perform search
  • LogDataWorker::doSearch() - Background search
  • LogDataWorker::searchFinished signal emitted
  • MainWindow::onSearchFinished() - Handle results
  • LogMainView::setSearchResults() - Highlight matches

3. Navigation Operations Flow

  • User clicks "Scroll to Line" button
  • MainWindow::KScrollToLine() called
  • QInputDialog opens for line number input
  • Line number validation performed
  • LogMainView::scrollToLine() - Scroll to line
  • LogMainView::setHighlightedLine() - Highlight line

4. Background Processing Flow

  • LogDataWorker thread starts
  • LogDataWorker::parseDataBlock() - Parse file blocks
  • LogDataWorker::findNextLineFeed() - Find line boundaries
  • LogDataWorker::loadLineContent() - Load line data
  • Progress signals emitted every 1000 lines
  • UI updated via signal-slot mechanism
